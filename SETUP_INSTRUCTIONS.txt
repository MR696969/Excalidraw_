# Excelidraw - Collaborative Drawing Application

A real-time collaborative drawing application built with Next.js, WebSockets, and PostgreSQL.

## 🚀 Features

- Real-time collaborative drawing
- User authentication and rooms
- WebSocket-based communication
- PostgreSQL database with Prisma ORM
- Modern UI with Tailwind CSS

## 🛠️ Tech Stack

- **Frontend**: Next.js 15, React 19, Tailwind CSS
- **Backend**: Node.js, Express, WebSocket (ws)
- **Database**: PostgreSQL with Prisma ORM
- **Authentication**: JWT tokens
- **Build Tool**: Turbo (monorepo)

## 📋 Prerequisites

Before running this project, make sure you have the following installed:

1. **Node.js** (version 18 or higher)
2. **pnpm** (package manager)
3. **PostgreSQL** (version 17 recommended)

### Installing Prerequisites

#### Node.js
Download from [nodejs.org](https://nodejs.org/) or use nvm:
```bash
nvm install 18
nvm use 18
```

#### pnpm
```bash
npm install -g pnpm
```

#### PostgreSQL
Download from [postgresql.org](https://www.postgresql.org/download/) or use your system's package manager.

## ⚙️ Setup Instructions

### 1. Clone the Repository
```bash
git clone <your-repo-url>
cd week-22-excelidraw
```

### 2. Install Dependencies
```bash
pnpm install
```

### 3. Database Setup

#### Create PostgreSQL Database
```sql
-- Connect to PostgreSQL as superuser (usually 'postgres')
-- Create the database:
CREATE DATABASE "chat-app";
```

#### Configure Environment Variables
Create a `.env` file in the `packages/db/` directory:

```env
DATABASE_URL="postgresql://postgres:your_password@localhost:5432/chat-app"
```

**Note**: Replace `your_password` with your actual PostgreSQL password.

### 4. Generate Prisma Client
```bash
cd packages/db
npx prisma generate
```

### 5. Run Database Migrations
```bash
cd packages/db
npx prisma migrate reset --force
```

### 6. Start Development Servers
```bash
pnpm dev
```

This will start all services:
- **Web App**: http://localhost:3000
- **Excelidraw Frontend**: http://localhost:3001
- **HTTP Backend**: http://localhost:3002
- **WebSocket Backend**: ws://localhost:8080

## 🐛 Troubleshooting

### Common Issues

1. **Port conflicts**: If ports 3000-3002 or 8080 are in use, the apps will automatically use the next available ports.

2. **Database connection errors**: Ensure PostgreSQL is running and the DATABASE_URL is correct.

3. **Prisma client errors**: Run `npx prisma generate` after any schema changes.

4. **Build errors**: Make sure all dependencies are installed with `pnpm install`.

### Database Issues
```bash
# Check database connection
cd packages/db
npx prisma migrate status

# Reset database (⚠️ This will delete all data)
npx prisma migrate reset --force
```
